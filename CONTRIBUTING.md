# Contributing

## Project overview

This package is a platform-independent fork of `@cloudflare/workers-oauth-provider`. The goal is to remove Cloudflare-specific runtime dependencies while keeping the core OAuth provider close to upstream for easier future merges.

## Repository structure

- `src/oauth-provider.ts`: upstream-ported core implementation. Keep changes minimal and migration-focused.
- `src/next.ts`: Next.js App Router helpers.
- `src/oidc/`: OIDC discovery and ID token verification helpers.
- `__tests__/`: Vitest suites and test setup.
- `playground/`: local integration playground.

## Setup

- Use Node.js 24 or newer.
- Install dependencies with `npm install`.

## Commands

| Command              | Purpose                         |
| -------------------- | ------------------------------- |
| `npm run build`      | Build the library with `tsdown` |
| `npm run check`      | Run typecheck and tests         |
| `npm run typecheck`  | Run TypeScript without emitting |
| `npm run test`       | Run the Vitest suite once       |
| `npm run test:watch` | Run Vitest in watch mode        |
| `npm run prettier`   | Format the repository           |

## Code standards

- TypeScript is strict-mode.
- Public methods and interfaces should have JSDoc.
- Prefer `PascalCase` for types, `camelCase` for values, and `SCREAMING_SNAKE_CASE` for constants.
- Use `Impl` for internal implementation suffixes and `Options` for configuration interfaces where that pattern already exists.
- Preserve the existing PImpl wrapper pattern around `OAuthProvider`.
- Maintain compatibility with both supported handler styles.

## Security notes

Treat all OAuth and token-handling changes as security-sensitive.

- Tokens and authorization codes are stored as SHA-256 hashes.
- Sensitive props are encrypted with AES-GCM.
- PKCE is required for public clients.
- Scope downscoping and audience handling must remain standards-compliant.

When behavior is unclear for MCP use cases, consult the latest published MCP authorization spec before changing code.

## Testing

Tests use Vitest and the custom mocks in `__tests__/setup.ts`.

Coverage includes:

- metadata discovery
- authorization code with PKCE
- token exchange and refresh
- client registration
- grant management and revocation
- validation and error handling
- resource-aware audience behavior

For behavior changes, update or add tests in the relevant suite:

- `__tests__/oauth-provider.test.ts`
- `__tests__/next.test.ts`
- `__tests__/oidc.test.ts`

Run `npm run check` before finishing.

## Changesets and release workflow

Add a changeset for public API changes and user-visible bug fixes:

```bash
npx changeset
```

CI validates the repository with build, typecheck, tests, and formatting checks.

## Generated files

- `dist/` is generated output. Do not hand-edit it.
- `package-lock.json` is generated by npm. Do not hand-edit it.
